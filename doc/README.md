# 我的日记本（My Diary Story）

一个纯本地运行的网页日记本应用，无需服务器，无需网络，数据保存在你电脑上的一个真实 JSON 文件中，**任何支持的浏览器都可以打开同一份数据**。

## 功能介绍

### 多日记本管理
- 创建多个独立的日记本，每本可以起名字、上传封面图片
- 首页以网格卡片形式展示所有日记本，显示封面图、名称、条目数量和最后更新日期
- 支持编辑日记本名称和封面，支持删除整本日记本（含其中所有条目）

### 日记条目
- 每个日记本下可以创建无限多篇日记条目
- 每篇日记支持：标题（可选）、正文内容、插入多张图片
- 自动记录**创建时间**和**最后编辑时间**，时间戳永久保留
- 条目列表按创建时间倒序排列，显示标题、内容摘要、第一张图片缩略图

### 跨浏览器数据共享
- 数据保存在你指定的一个真实 `.json` 文件中
- Chrome、Edge、Safari 打开同一个数据文件，看到的数据完全一致
- 换电脑只需复制这一个文件

### 数据管理
- 查看当前数据文件信息（文件名、大小、日记本/条目/图片数量）
- 一键**下载备份副本**（额外备份，不影响原数据文件）
- 从备份文件**导入数据**恢复
- **切换数据文件**（使用不同的数据文件）
- 清空当前数据文件中的所有数据

### 界面
- 深色 / 浅色主题切换（右上角月亮/太阳按钮）
- 主题偏好自动保存
- 响应式布局，适配不同屏幕尺寸
- 快捷键：`Ctrl+S`（Windows）/ `Cmd+S`（Mac）在编辑页快速保存
- 每次保存后导航栏显示"已保存"提示

## 如何打开使用

### 第一次使用

1. 将 `my_diary_story` 文件夹放到你电脑任意位置
2. 用 **Chrome 或 Edge** 打开其中的 `index.html`
3. 应用会显示欢迎页，点击 **"新建数据文件"**
4. 在系统文件对话框中选择保存位置（建议放在文档目录或云同步文件夹），命名为 `diary_data.json`
5. 点击保存，即可开始使用

> 建议将数据文件放在 iCloud Drive / OneDrive / Dropbox 等云同步目录中，这样文件本身会自动备份到云端。

### 后续使用

每次打开 `index.html`，应用会自动识别上次使用的文件并显示 **"重新打开上次的文件"** 按钮，点击并授权即可直接进入。

> 授权弹窗是浏览器的安全机制，每次打开新标签页都需要点击一次确认。

## 应用架构

```
my_diary_story/
├── index.html    # 页面结构（所有视图的 HTML 骨架）
├── style.css     # 样式（浅色/深色主题、响应式布局）
├── app.js        # 所有业务逻辑（数据层 + UI 层）
└── README.md     # 本文档
```

### 数据存储架构（v2）

```
diary_data.json（你电脑上的真实文件）
├── books[]      —— 日记本列表
├── entries[]    —— 所有日记条目（文字内容 + 元数据）
└── images[]     —— 所有图片（base64 编码，内嵌在 JSON 中）
```

- 所有数据（文字 + 图片）都存在**同一个 JSON 文件**中
- 每次编辑保存后，整个文件会被重新写入
- 图片在上传时自动压缩（宽度上限 1200px，JPEG 质量 82%）
- 主题偏好（深色/浅色）存在浏览器 localStorage 中，与数据文件无关

### 数据模型

**日记本（DiaryBook）**
```json
{
  "id": "唯一ID（时间戳+随机串）",
  "name": "日记本名称",
  "coverImageId": "封面图片ID",
  "createdAt": "2024-01-01T12:00:00.000Z",
  "updatedAt": "2024-01-15T09:30:00.000Z"
}
```

**日记条目（DiaryEntry）**
```json
{
  "id": "唯一ID",
  "bookId": "所属日记本ID",
  "title": "日记标题",
  "content": "正文内容",
  "imageIds": ["图片ID1", "图片ID2"],
  "createdAt": "2024-01-10T20:00:00.000Z",
  "updatedAt": "2024-01-10T21:30:00.000Z"
}
```

**图片（内嵌在 images 数组）**
```json
{
  "id": "img_唯一ID",
  "data": "base64编码的图片数据",
  "mimeType": "image/jpeg",
  "size": 102400
}
```

## 数据存储位置

数据文件存在**你自己选择的位置**，完全由你掌控：

- 你在首次使用时通过系统文件对话框选择的路径
- 可以是桌面、文档文件夹、iCloud Drive、Dropbox 等任意位置
- 文件名就是你保存时起的名字（默认建议 `diary_data.json`）

### 与旧版（v1）的区别

旧版数据存在浏览器内部（localStorage/IndexedDB），新版存在真实文件中：

| 特性 | 旧版（v1） | 新版（v2） |
|---|---|---|
| 存储位置 | 浏览器内部 | 你指定的文件 |
| 跨浏览器 | 不支持 | 支持 |
| 清除浏览器数据时 | 数据丢失 | 数据不受影响 |
| 换电脑 | 需导出/导入 | 直接复制文件 |

## 数据备份与恢复

### 日常备份（推荐）

将数据文件放在 **iCloud Drive / OneDrive / Dropbox** 等云同步目录中，文件会自动同步备份到云端，无需手动操作。

### 手动下载备份副本

1. 点击右上角 **"数据管理"** 按钮
2. 在"下载备份副本"卡片中点击 **"⬇ 下载备份副本"**
3. 浏览器会下载一个 `diary_backup_YYYYMMDD.json` 文件
4. 将此文件保存到其他位置作为额外备份

### 从备份恢复

1. 进入"数据管理"页
2. 在"从备份文件导入"卡片中点击 **"⬆ 选择备份文件导入"**
3. 选择备份文件，确认后数据会恢复到当前数据文件中

> 注意：导入会覆盖当前数据文件中的全部数据，操作前建议先下载一份备份副本。

## 在不同电脑或浏览器上使用

### 场景一：换电脑

1. 将 `diary_data.json`（数据文件）和 `my_diary_story` 文件夹（应用文件）复制到新电脑
2. 用 Chrome 或 Edge 打开 `index.html`
3. 欢迎页点击 **"打开已有数据文件"**，选择 `diary_data.json`
4. 授权读写权限后，所有数据立刻可用

### 场景二：同一台电脑，换浏览器（如从 Chrome 换到 Edge）

1. 在新浏览器中打开 `index.html`
2. 欢迎页点击 **"打开已有数据文件"**，选择同一个 `diary_data.json`
3. 数据完全一致，因为是同一个文件

### 场景三：多台电脑同步使用

将数据文件放在云同步目录（iCloud Drive / OneDrive / Dropbox）中：

```
~/iCloud Drive/
└── diary_data.json   ← 放在这里
```

每台电脑打开各自的 `index.html`，都选择这个云同步目录中的文件。文件修改后会自动同步，实现类似"云同步"的效果。

> **注意**：避免同时在多台电脑上编辑，可能导致云同步冲突（文件互相覆盖）。

### 场景四：使用多个数据文件

你可以创建多个数据文件，分别用于不同用途（如工作日记、旅行日记）：

- 欢迎页"新建数据文件"创建新文件
- 数据管理页"切换数据文件"切换到不同文件

## 注意事项

### 文件保存机制
- 每次点击"保存"、创建/删除日记本、创建/删除条目，都会自动将整个数据文件重新写入
- 导航栏会出现短暂的"已保存"提示
- 建议不要同时在两个浏览器标签页中打开同一个数据文件，可能导致数据覆盖

### 文件大小
- 纯文字日记：文件通常很小，100 篇日记 < 1MB
- 含图片：每张图片压缩后约 100-500KB，100 张图片约 10-50MB
- JSON 文件过大（> 100MB）时读写速度会变慢，建议定期清理不需要的图片

### 图片
- 上传时自动压缩，宽度超过 1200px 的图片等比缩小
- JPEG 质量：82%；PNG 保持无损格式
- 图片数据内嵌在 JSON 文件中（base64 编码）

### 不要删除数据文件
- 数据文件就是你的所有日记，删除文件即永久删除所有数据
- 定期将数据文件复制到其他位置或云盘作为备份

### 隐私说明
- 本应用完全不联网，没有任何数据上传到服务器
- 所有数据仅存在你指定的本地文件中
- 应用文件（HTML/CSS/JS）不包含任何追踪代码或外部依赖
